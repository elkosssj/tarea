Ejercicio 11 de 22

(ab)⁺ | c

/ (ab)⁺ | c /

Una o más repeticiones del bloque "ab", o bien la cadena "c". El operador | es unión. El ⁺ exige al menos una repetición.

1.1 Lenguaje Regular de Aceptación — Definición formal ›
LENGUAJE FORMAL
Lr11 = { (ab)n | n ≥ 1 } ∪ { c }
= { ab, abab, ababab, abababab, …, c }
Σ = { a, b, c }
¿Por qué? El operador (ab)⁺ genera la cadena "ab" repetida al menos una vez. El | agrega la alternativa "c". Las dos familias son completamente distintas: cadenas de longitud par que alternan a-b, y la cadena "c" sola de longitud 1.
1.2 Autómatas Finitos — AFND-ε · AFND · AFD ›
Estado final (doble círculo)
Estado inicial
ε = transición vacía
Q \ ΣεabcFINAL
→q0{q1, q5}∅∅∅0
q1∅{q2}∅∅0
q2∅∅{q3}∅0
q3{q1, q4}∅∅∅0
q4 ✓∅∅∅∅1
q5∅∅∅{q4}0
Construcción Thompson: La unión crea dos caminos desde q0 con ε. El bucle q3→q1 con ε implementa el ⁺. q3 también llega a q4 (final) con ε para aceptar tras cualquier repetición.
Q \ ΣabcFINAL
→q0{q1}∅{q2}0
q1∅{q2}∅0
q2 ✓{q1}∅∅1
Sin ε: Se calculan las clausuras-ε. q0 va con "a" o con "c". q2 puede volver a q1 con "a" para repetir el ciclo ab.

AFD mínimo: A={q0}, B={q1}, C={q2}=final. qT es el estado trampa (error). Cada estado tiene exactamente una transición por símbolo.

Ejercicio 15 de 22

(a | b*)c⁺

/ (a|b*)c⁺ /

(a | b*): la letra "a" o bien cero o más "b". Como b* incluye ε, la cadena "c" sola es aceptada. Seguido de c⁺: al menos una "c".

1.1Lenguaje Regular de Aceptación›
LENGUAJE FORMAL
Lr15 = { ack | k≥1 } ∪ { bmck | m≥0, k≥1 }
= { c, ac, bc, cc, acc, bcc, bbc, accc, bbbc, … }
Σ = { a, b, c } — "c" ∈ L porque b* con m=0 produce ε
1.2Autómatas Finitos›
Q\ΣεabcFINAL
→q0{q1, q2}∅∅∅0
q1∅{q3}∅∅0
q2{q3}∅{q2}∅0
q3∅∅∅{qF}0
qF ✓∅∅∅{qF}1
Q\ΣabcFINAL
→q0{q1}{q0}{q2}0
q1∅∅{q2}0
q2 ✓∅∅{q2}1
Q\ΣabcFINAL
→ABAC0
BqTqTC0
C ✓qTqTC1
qTqTqTqT0
A={q0}, B={q1}, C={q2}=final. A tiene auto-lazo con 'b' porque b* se queda en el mismo estado inicial. Después de leer 'c' ya no se puede leer 'a' ni 'b'.
